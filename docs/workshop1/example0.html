HTTP

Most web content is transported using the HTTP protocol. The HTTP protocol is a server-client protocol in that there is a single server and arbitrarily many clients. When a client makes a request to the server, the server replies with an appropriate response.

HTTP Requests

  • A method (GET, POST, PUT, DELETE).
  • A URL (e.g. /foo/bar/baz)
  • A set of metadata headers (e.g. Date, Content-Type, etc)
  • A body (arbitrary data)

HTTP Responses

  • A status codeto indicate success or error (e.g 404).
  • A set of metadata headers (e.g. Date, Content-Type, etc)
  • A body (arbitrary data)

Resources and CRUD

The URL of a HTTP request is intended to represent set of nested resources, for example files within a nested directory. The method of an HTTP request is meant to represent the action that is taken on a resources.

  • POST (create a file)
  • GET (read a file)
  • PUT (update a file)
  • DELETE (delete a file)

REST API

A REST API generalizes the notion of creating, reading, updating, and deleting a file to arbitrary data.

Data (in a HTTP body) may be transmitted in many different formats.

  • plaintext
  • html
  • JSON
  • ...etc

HTML

Try making a following GET request to the endpoint '/' on the server 'ugcs.caltech.edu'. This can be done simply by typing 'ugcs.caltech.edu/' into your browser.

When you enter a URL into the search bar, the web browser sends the following HTTP request to the 'ugcs.caltech.edu' server.


GET /

The 'ugcs.caltech.edu' server should return a response that looks like the following... try it out!


HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Content-Length: 702

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="style.css">
    <title>UGCS</title>
</head>
<body>
    <nav>
        <h1><a href="/">UGCS</a></h1>
        <div>
            <a href="/leadership.html">Leadership</a>
            <a href="/calendar.html">Calendar</a>
        </div>
    </nav>
    <main>
        <h1>Undergraduate Computer Science Club</h1>
        <p>
            The mission of UGCS is to maintain a student-run computing environment that serves as sandbox for
            experimentation and innovation. Additionally, UGCS seeks to build an alumni network to help students
            navigate the internship and job search process.
        </p>
    </main>
</body>
</html>
